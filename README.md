# incident-tracker
Production-ready incident tracking service built with FastAPI and MySQL

##PURPOSE
This project demonstrates backend engineering fundamentals:
-REST API design with FastAPI
-Relational data modeling (MySQL)
-Data validation and error handling
-Production-minded structure

## Planned features
-Create and manage incident events
-Track severity and status
-Query incidents with filters
-Basic metrics and reporting
-CI/CD pipeline (GitHub Actions)

## Tech Stack
-Python (FastAPI)
-MySQL
-SQLAlchemy
-Docker
-GitHub Actions

##Status
In progress - foundational setup underway
